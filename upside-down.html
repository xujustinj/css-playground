First Section

You may have noticed that you loaded into the page on the bottom.

That is caused by a little bit of JavaScript in the onload function of the body element.

Second Section

The sections of this document are ordered from bottom to top using the column-reverse flex layout direction.

Inside each section, the text is still read from top to bottom by default. This can be changed by setting a column-reverse layout on the section container.

Third Section

If you inspect the HTML tree, you will see that the elements are in the "correct" order.

That is, the page is only upside-down in appearance.

Fourth Section

This approach is friendly to screen readers, which use the order of the HTML tree.

If your browser has a read-aloud feature, you can try it out for yourself.

Fifth Section

That said, users who interact with your site visually will be somewhat confused by the unintuitive direction of flow.

You should use upside-down layouts sparingly, and only when the design of the site makes it natural.
